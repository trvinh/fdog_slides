    RBH-based method

    Chen et al. (2007), Plos ONE

    The basic method for ortholog prediction is reciprocal best hits
    If B1 is the best hit of A1 in species B, and A1 is again the best hit of B1 in species A => A1 and B1 are orthologs
    The origin of fDOG

    HaMStR

    Hamstr is one of the apporoaches that are bases on bidirectional search

    HaMStR

    Starting from a profile HMM of a set of core orthologs, hamstr search for candidate sequences in protein set G
    Each candidate will then be searched agains the reference proteome F using Blast
    If the reciprocal criterion fulfilled, that candidate protein will be added into the group as a new ortholog.
    So, for hamstr, we need to provide the core ortholog group as an input. The core compilation can be done using other tools like OMA or OrthoMCL

    +


    fDOG

    Then, we had an idea to combine both the core compilation and the ortholog search into hamstr. At the beginning, we call this new tool as hamstr-oneseq. Then later, we renamed it into fdog.
    How fDOG works
    First step of fdog is the core compilation. The profile HMM of the seed sequence will be used to searched for similar sequences in a tree of the core taxa iteratively from the close relative to distant relatives of the reference species. The taxon rank can be defined by the user (e.g. only from genus to kingdom).
    Then, the candidate that maximizes the similarity in both the sequence and the feature architecture will be added in to the core group.
    This step will be repeated until the number of the core orthologs exceeded a pre-defined value (by default is 5)
    After having the core group, the normal hamstr search will be applied to search for ortholog in a set of search taxa
    Lastly, fdog calculates the fas scores for all orthologs against the seed protein. The final output can be visualized using PhyloProfile for further study.

    Benchmark method



    We used the Orthology Benchmarking Service to compare fdog with other tools.
    This is a public service for assessing the performance of different ortholog prediction approaches based on the evaluations of 4 different categories.
    Here we focused only on the first test, the species tree discordance test, and the last 2, the GO and EC test.
    Species tree discordance test: test for the congruence of inferred orthologs with a trusted reference tree
    Gene Ontology test & EC number test: evaluated orthology in terms of functional similarity by comparing the GO term and EC numbers between the query and ortholog sequences
    • Compared using Quest for Orthologs - Challenge 6
    This is the number of the pairwise orthologs taken into account for the comparison from different approaches.
    For fdog, we take the whole prediction result without any post filtering, as well as filtered the data by the forward, backward and the mean fas score
    in which only the pairwises that have the corresponding FAS score not less than 0.75 were considered.
    We then compared the 4 fdog ortholog sets with InParanoid and OMA-Pairs.
    EC conservation test: x-axis is the recall and y-axis is the precision of the evaluation. For the precision they used the average schlicker scores - a semantic similarity score for comparing sets of annotations such as GO terms or EC numbers
    Tree discordance test: Robinson-foulds distance: the sum of partitions of data that is implied by one tree but not by the other.
    How meaningful are the core ortholog groups

    ... in reflecting the species relationship

    Another aspect we want to check is to see how well the core ortholog groups are in term of explaining the species relationship of the core taxa. This figure shows the schematic reference species tree of the 78 QfO taxa.

    • Topology test using IQ-TREE

      (Topology rejected (-) when p-AU < 0.05, otherwise accepted (+))

    • 18,067 core groups with more than 3 sequences

    For this, we compared the topology of gene trees created from the proteins of each core group with the (pruned) reference species tree.
    This is an example of the topology test output from iq-tree. Here we focused on the p-value of the AU test. If the p-value less than 0.05, the topology of the gene tree is significantly different to the one of the species tree.
    We had in total 18,067 tests for all the core groups that have more than 3 sequences.
    AU-test: Approximately Unbiased test - uses multi-scale bootstrapping of the site-likelihoods obtained from fitting molecular or other data to a set of topologies, in order to identify the subset of topologies that are consistent with the data
    Approach Accepted Rejected
    No correction 11,707 (64.80%) 6,360 (35.20%)
    Benjamini + Hochberg (BH) correction 13,625 (75.41%) 4,442 (24.59%)
    Bonferroni correction 16,204 (88.69%) 2,043 (11.31%)
    This is the number of accepted (not significantly different) and rejected (significantly different) gene trees.
    Before correction for multiple testing problem, 65% of the gene trees was accepted. After corrected using BH correction and Bonferroni correction, the number of accepted trees increased up to 89%.
    Here we need to applied the multiple testing correction to reduce the false positive rate caused by the high number of multiple statistical tests.
    • Problem: if one test is performed at the 5% level and the corresponding null hypothesis is true, there is only a 5% chance of incorrectly rejecting the null hypothesis. However, if 100 tests are conducted and all corresponding null hypotheses are true, the expected number of incorrect rejections (also known as false positives or Type I errors) is 5. If the tests are statistically independent from each other, the probability of at least one incorrect rejection is 99.4%.
    • => Multiple testing corrections adjust p-values derived from multiple statistical tests to correct for occurrence of false positives
    • Bonferroni "punishes" all input p-values equally, whereas Benjamini-Hochberg (as a way to control the False Discovery Rate) "punishes" p-values accordingly to their ranking. If you have few tests, it should not make too much of a difference, but if you have many tests (as for example when you test all 20,000 human protein coding genes or even larger sets), it will make a difference. In this case Bonferroni will produce false negatives, in other words it will discard significant observations. Therefore usually I use Benjamini-Hochberg
    the mean of delta-log likelihood for accepted topologies is significant smaller than the one of rejected topologies
    (Mann-Whitney-Wilcoxon p-value < 2.2e-16)

    An use-case: Where does the archaea locate

    Eme et al. (2017), Nat. Reviews Microbiology
    1. The well-known three-domains tree of life
    2. Two-domains tree of life
    3. Eukaryota branches as sister to TACK superphylum
    4. Eukaryota originated from within (or as sister to) the Asgard archaea
    As an use-case, we try to resolve the debated position of the archaea in the species tree of life using the orthologs predicted by fdog.
    This is a schematic representation of our understanding of the relationships between eukaryotes and archaea over the past 40 years.
    Here we have the well-known 3-domains tree of life in the figure a; and the others are the 2-domains tree of life, where eukaryota was placed within the archaea.
    Starting from 11 asgardarcheaota => got 343 core ortholog groups => expanded in 732 archaea, 48 eukaryota and 23 bacteria => subselected 68 representatvie taxa that cover the diversity of archaea => reconstructed the species tree
    • 48 core genes (FAS >= 0.75, present in at least 50% taxa of each superkingdom)
    • Super-alignment of 139 sequences, 13,060 aa in length
    • Tree reconstruction using IQ-TREE
      • -m MFP: auto identify best fitting model with ModelFinder => LG+R9
      • -bb 1000: ultrafast bootstrap approximation with 1000 replicates
      • -bnni: reduce the risk of overestimating branch supports with UFBoot
    For the tree reconstruction, we identified 48 core genes that have FAS >= 0.75 & present in at least 50% of the taxa in each supterkingdom (the one in the red box on this profile plot).
    From that, we got a super-alignment for the 139 taxa with the lenght of about 13,000 amino acids.
    Then, we used IQ-tree to reconstruct the tree using these parameters:

    LG substitution with FreeRate model

    This is the ML tree we got from IQ tree. The bootstrap support values are shown in the internal branches, where red are branches that have the bootstrap values between 50 and 99; blue are less then 50 and the others are 100
    The tree was rooted using bacteria as the outgroup.
    Overall, we had a very well resolved archaea clade. But some of the eukaryota or bacteria species still have a wrong positions, such as HUMAN is placed as sister to chimp and gorrila. But, since we start from the asgard core genes, it is difficult to resolve such a far away nodes.

    To better compare with the different species tree of life hypotheses, I re-draw the previous tree with this schematic tree.
    Clearly,
    • it supports the three-domains tree of life (Williams et al. 2020 and Raymann et al. 2015 => 2 domains tree of life)
    • Asgard are polyphyletic (same as Williams et al. 2020) and are sister clade of the other archaea
    • Euryarchaeota are monophyletic (while Williams et al. 2020 and Raymann et al. 2015 showed polyphyletic => used bayesian tree reconstruction)
    • Korarchaeota placed within TACK superphylum (Raymann et al. 2015: outside of TACK, as sister of Euryarchaeota)
